What the DOM actually is
1. A programming interface for HTML and XML documents
2. A live representation of the page in memory
3. Made up of objects, not just text

Each HTML element becomes an object you can:
--> Read
--> Change
--> Add to
--> Remove
--> Listen to for events

The DOM is what lets JavaScript do things like:
1. Change text or HTML
2. Update styles
3. React to user actions (clicks, typing, scrolling)
4. Create dynamic content
Example:
	document.querySelector("h1").textContent = "Hi there!";
	--> Access the DOM
	--> find h1 tag
	--> change the text
	
Shadow DOM
----------
Shadow DOM is a way to create isolated, encapsulated DOM trees inside elements, protecting their structure and styles from the rest of the page.
It lets developers:
1. Encapsulate HTML, CSS, and behavior
2. Prevent styles and structure from leaking in or out
3. Build reusable, self-contained components
"DOM inside another DOM"
The #shadow-root is the Shadow DOM.

1. Shadow Host: The regular DOM node that the shadow DOM is attached to.
2. Shadow Root: The root node of the shadow tree. 
3. Shadow Tree: The hidden DOM tree inside the shadow DOM.
4. Shadow Boundary: The invisible line where the shadow DOM ends and the regular (light) DOM begins.

you can automate Shadow DOM elements, but not exactly like normal DOM
Example: 
	<my-component>
	  #shadow-root
		<button>Click</button>
	</my-component>
	if you try to click the button then the browser blocks direct traversal into #shadow-root.
	if you want to access shadow DOM elements then you must explicitly enter the shadow root.
steps to access the shadow DOM elements
1. You must get the shadow host first,
2. then access its shadowRoot, 
3. then locate the element and perform action
There may be nested shadow DOM's also
